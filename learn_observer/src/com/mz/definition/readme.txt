观察者结构的简单定义包
观察者模式中有两方:
    1、被观察者（主题）
    2、观察者（通知者）

观察者模式典型实现方式：
    1、针对模式中出现的双方定义2个接口：
         a、观察者（通知）接口；
         b、被观察者（主题）接口；

    2、分别定义接口的3个实现类：
        a、两个观察者(通知)对象分别实现观察者接口；
        b、被观察者(主题类)对象实现被观者接口；

    3、主题类注册自己需要通知的观察者

    4、主题类某个业务逻辑发生时通知观察者对象，每个观察者执行自己的业务逻辑


关键点：
　　　　1、针对观察者与被观察者分别定义接口，有利于分别进行扩展。
　　　　2、重点就在被观察者的实现中：
　　　　　　（1）定义观察者集合，并定义针对集合的添加、删除操作，用于增加、删除订阅者（观察者）
　　　　　　（2）定义通知方法，用于将新情况通知给观察者用户（订阅者用户）
　　　　3、观察者中需要有个接收被观察者通知的方法。
